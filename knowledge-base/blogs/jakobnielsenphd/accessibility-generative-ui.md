# Accessibility Has Failed: Try Generative UI = Individualized UX

**Date:** 2024-02-29

**Source:** [Original Article](https://jakobnielsenphd.substack.com/p/accessibility-generative-ui)

---

## Summary

Jakob Nielsen argues that traditional accessibility methods have failed to meaningfully improve computer usability for disabled users over the past 30 years. He proposes that second-generation generative UI--which creates customized interfaces for each user at runtime--offers a more scalable solution than conventional accessibility approaches.

## The Failure of Traditional Accessibility

Nielsen contends that accessibility has been ineffective by three key measures: difficulty of use, productivity when performing tasks, and user satisfaction. Despite three decades of effort, computers remain "difficult, slow, and unpleasant" for disabled users.

Two primary reasons explain this failure:

1. **Cost prohibitiveness**: Companies cannot afford comprehensive accessibility implementation across all disability types, so many either ignore it entirely or meet only minimum checklist requirements without proper usability testing.

2. **Inherent limitations**: "Linear auditory user interfaces cannot adequately represent two-dimensional graphical interfaces," particularly disadvantaging blind users who must listen sequentially rather than scan visually.

## Important Exceptions

Two large user groups can be effectively served with current methods:

**Elderly Users (75+)**: The United States has approximately 23 million seniors with combined net worth around $23 trillion. These users benefit from simplified navigation and explanations.

**Low-literacy Users**: Roughly 40% of American adults qualify as low-literacy (approximately 100 million people). Writing at an eighth-grade level serves this demographic effectively.

## The Generative UI Solution

### First-Generation vs. Second-Generation

**First-generation generative UI** uses AI during design to create draft interfaces that human designers then refine before deployment. All users receive identical UIs.

**Second-generation generative UI** (projected within five years) generates fresh interfaces each time users access applications. Different users receive drastically different designs optimized for their individual needs, capabilities, and learning progress.

### How It Works

Rather than forcing blind users through screen readers that linearize 2D layouts, generative UI directly creates optimized auditory interfaces from underlying data. Text can be adjusted to reading levels, and complexity can scale from beginner to expert modes.

### Impact on Design Work

This shift fundamentally changes UX professional roles. Instead of designing specific interfaces, designers will "specify rules and heuristics the AI uses to generate the UI." Nielsen compares this loss of pixel-perfect control to the responsive web design transition, where designers adapted to variable screen sizes rather than fighting them.

## Reader Responses

The article generated significant critical discussion. One commenter argued that Nielsen's piece "simultaneously diminishes significant accessibility gains" and "flatters organizations that refused to invest" while ignoring the complex work required to ensure AI consistency and control over outcomes.

Another raised practical concerns about context awareness, noting that "personalized interfaces create challenges when multiple users share devices" and questioned whether universal design principles might address these implications more effectively.
