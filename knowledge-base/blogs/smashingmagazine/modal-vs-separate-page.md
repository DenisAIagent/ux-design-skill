# Modal vs. Separate Page: UX Decision Tree

**Date:** March 19, 2026
**Author:** Vitaly Friedman
**Source:** https://www.smashingmagazine.com/2026/03/modal-separate-page-ux-decision-tree/

## When Modals Help
- High-priority, short interactions
- Destructive action confirmations
- Quick selections or alerts
- Maintaining user context and scroll position

## When to Avoid Modals
- Error messages
- Feature notifications
- Onboarding experiences
- Complex multi-step workflows

## Decision Framework (Ryan Neufeld)
1. Does the user need background context?
2. Is the task simple or complex?
3. Will users reference underlying data?
4. Should it be modal or nonmodal?

## For Repeated Tasks
"Expandable sections or in-place editing often work better — they keep the task anchored to the current screen."
