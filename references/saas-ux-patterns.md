# SaaS UX Patterns — Comprehensive Reference

Sources: @clea_ux (92 transcriptions), Masterclass SaaS Growth (6 modules)

## 1. ONBOARDING

### Core Principle
L'onboarding n'est pas un tutoriel, c'est un **raccourci vers la valeur**. Objectif : amener l'utilisateur au "Aha Moment" en moins de 2 minutes.

### Patterns Clés
- **Jamais de dashboard vide** — Toujours proposer une action immédiate ("Créer votre premier lien")
- **Barre de progression émotionnelle** — Pas "Étape 2/4" mais "Bravo, plus que 2 étapes et vos candidats peuvent se projeter dans votre entreprise"
- **Micro-victoires** — Célébrer chaque succès (confettis, messages positifs)
- **Onboarding contextuel par feature** — Au premier clic sur chaque feature importante, mini-onboarding dédié (3-5 étapes max), déclenché une seule fois
- **Chemin le plus court** — Chaque écran qui n'amène pas vers le Aha Moment → supprimer. Chaque config qui peut attendre → après. Chaque info demandée trop tôt → supprimer
- **Déblocage progressif** — L'utilisateur débloque les features au fur et à mesure de sa progression
- **Data fictive pré-remplie** — Montrer le produit "en action" avec des données exemples

### Les 5 Fenêtres d'Intention
1. **Juste après l'inscription** — Motivation maximale → lancer vers le Aha Moment
2. **Juste après le Aha Moment** — Valeur perçue → proposer upgrade, invitation d'équipe
3. **Quand il fait une erreur** — Frustré mais attentif → guider vers la solution en 1 action
4. **Quand il abandonne en cours de route** — Cerveau déteste l'inachevé → "Qu'est-ce qui t'a arrêté ?"
5. **Retour après longue absence** — Signal fort mais fragile → résumé + action claire

## 2. LANDING PAGE

### Hero Section
- Proposition de valeur claire + aperçu visuel du produit
- Jouer sur la **frustration émotionnelle** : "Ne perdez plus jamais une vente à cause d'un lien cassé" > "Gérez tous vos liens"
- Sous-titre = expliquer le "comment" de la magie
- < 10 secondes pour convaincre (effet de Halo)

### Réduction de Friction
- Éléments de réassurance sous le CTA ("Sans carte bancaire", "Gratuit 14 jours")
- CTA = décrire l'APRÈS : "Commencer gratuitement" > "S'inscrire"

### Section Pricing (Effet de Contraste)
- **Ancre haute** — Option très chère (199€) à côté de l'option cible (79€)
- Signal social sur le plan cible ("Le plus populaire")
- Plan d'entrée = montrer ce qui manque pour justifier le plan du milieu

### Section Comparatif
- Montrer uniquement les critères où vous gagnez
- Croix rouge = plus d'impact qu'un paragraphe

### Fonctionnalités
- **Avant/Après** plutôt que liste de features

## 3. RÉTENTION & FIDÉLISATION

### Distinction Critique
- **Rétention** = fonctionnelle (il reste). **Fidélisation** = émotionnelle (il reste, il aime, il recommande)

### Les 5 Leviers de Fidélisation
1. **Valeur prouvée** — "Ce mois-ci vous avez économisé 6h de travail"
2. **Personnalisation progressive** — Plus il configure, plus partir = tout recommencer
3. **Moments de Delight** — Petites attentions inattendues
4. **Considération** — "On a entendu vos retours, voici ce qui a changé"
5. **Communauté** — Transformer le produit en "tribu"

### Page 404 comme Tunnel de Vente
- Guide vers fonction phare + conciergerie + lead magnet

## 4. NOTIFICATIONS — 5 Erreurs à Éviter
1. Notifier pour rappeler qu'on existe → aucune valeur
2. Notifier trop souvent → tout est ignoré
3. Notifier sans action possible → opportunité gâchée
4. Notifier sur des sujets hors-sujet → personne ne s'en soucie
5. Ne pas donner le contrôle → tout est désactivé

**Règle d'or :** La notification idéale est celle que l'utilisateur attendait sans le savoir.

---

## 5. UX WRITING & MICRO-COPY

- Le micro-copy enlève un doute avant qu'il se forme
- Boutons = décrire l'APRÈS le clic : "Sauvegarder et générer l'analyse" > "Sauvegarder"
- Placeholders = guider ("8 caractères minimum"), qualifier ("contact@monentreprise.com")
- Messages d'erreur = POURQUOI bloqué + action immédiate. Jamais un point final, toujours une étape
- Confirmation = confirmer + suggérer la prochaine étape
- Effet de cadrage : "Moins d'1€/jour" > "29€/mois" (même prix, perception différente)

---

## 6. DASHBOARD & EMPTY STATES

- Si tu ne sais pas quoi mettre sur le dashboard → tu n'en as pas besoin
- Pour un MVP : skip le dashboard, aller directement dans l'action
- Jamais "Pas de données" sans CTA → toujours proposer de créer le premier élément
- Boutons "Créer" → toujours préciser QUOI

---

## 7. BARRE DE RECHERCHE

- Proactive dès 2 lettres (résultat avec photo/info)
- Recherches populaires avant de taper
- Actions rapides dans les résultats (comme Spotlight)
- Contextuelle selon l'action en cours
- Zéro résultat → features populaires + vote roadmap + correction d'intention

---

## 8. ANALYTICS vs UX

- Analytics = OÙ les utilisateurs partent. UX = POURQUOI
- 4 actions : session replays, 2-3 actions AVANT le départ, micro-survey, contacter les partis
- KPIs : fréquence de retour (3+/semaine = habitude), nombre d'actions, point d'accrochage
- Erreurs : inscriptions sans activation, taux global sans décomposition, CAC sans LTV (ratio min 3, idéal 5+)

---

## 9. GAMIFICATION

- Compteur de valeur accumulée, déblocage progressif, complétion sociale
- Récompense inattendue (au 10e usage → template exclusif)
- Statut invisible (accès anticipé pour les plus actifs)
- **5 Leviers Anti-Churn** : effort visible, micro-tâches, erreurs dé-dramatisées, continuité de reprise, progression dans le temps

---

## 10. FEEDBACK SYSTEM (3 Types)
1. **Confirmation** — "Est-ce que ça a marché ?"
2. **Direction** — "Qu'est-ce que je fais maintenant ?"
3. **Progression** — "Est-ce que j'avance ?"

## 11. PATTERNS ADDITIONNELS

### Mobile UX
- Ne PAS concevoir comme la version responsive du web
- Une intention par écran, accessibilité au pouce, feedback sensoriel

### Page de paiement
- Meilleur moment psychologique → célébrer, créer l'anticipation, donner une action immédiate

### Écrans de chargement
- Transformer l'attente passive en découverte : tips produit, découverte de features

### Mode Focus
- Pendant les tâches concentrées : cacher menus, notifications, options non pertinentes

### Parcours utilisateur
- Séparer navigation landing page et web app (www vs app)
- "Here is the next step" > "What do you want to do?"

### Processus de création SaaS
1. Tester l'idée (pas de dev) 2. MVP rapide 3. UX sur le réel 4. Marketing 5. Itération continue

### Cycle de vie utilisateur
Acquisition → Conversion → Activation → Rétention → Fidélisation

### Les 3 Piliers d'un SaaS qui Scale
Développement + Marketing + UX/UI Design — Les 3 sont indispensables, l'UX est le plus sous-estimé

### UX vs UI Priority
UX first, UI after. Un produit intuitif mais moche convertit. Un produit joli mais confus fait fuir.
